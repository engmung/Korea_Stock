# Korea_Stock 시스템 테스트 가이드

단계별로 따라하면서 파이프라인 전체를 검증하는 테스트 플랜입니다.

---

## 사전 준비

```bash
# 1. 패키지 설치
pip install -r requirements.txt

# 2. 서버 실행
python main.py
```

서버가 `http://localhost:8003` 에서 정상 기동되면 다음으로 진행합니다.

---

## Step 1. 설정 확인

**목적**: `.env` 설정이 올바르게 로드되는지 확인

```
GET http://localhost:8003/config
```

✅ 확인할 것:
- `llm_provider`, `llm_model`이 의도한 값인지
- 스케줄링 주기 값이 맞는지

```
GET http://localhost:8003/
```

✅ 확인할 것:
- 서버 상태 메시지 + LLM 정보가 정상 출력되는지

---

## Step 2. Notion DB 연결 확인

**목적**: 4개 DB가 정상적으로 연결되는지 확인

```
GET http://localhost:8003/channels
GET http://localhost:8003/queue
GET http://localhost:8003/reports
GET http://localhost:8003/opinions
```

✅ 확인할 것:
- 각 API가 `{"status": "success", ..., "total": 0}` 형태로 응답하는지
- 에러 없이 빈 배열이 반환되면 정상

⚠️ 에러가 나면:
- `.env`의 `NOTION_API_KEY` 확인
- Notion Integration이 4개 DB에 모두 공유(연결)되어 있는지 확인

---

## Step 3. 채널 등록 (Notion에서 직접)

**목적**: 채널 DB에 테스트용 채널 1~2개를 수동 등록

### Notion 채널 DB에 직접 추가:

| 채널명 | URL | 키워드 | 활성화 |
|--------|-----|--------|--------|
| 한국경제TV | `https://www.youtube.com/@한국경제TV` | 주식 | ✅ |

> 💡 키워드는 해당 채널에서 찾고 싶은 영상 제목에 포함된 단어입니다.
> 유튜브 채널 페이지에 가서 최근 영상 제목을 보고 적절한 키워드를 설정하세요.

### 등록 확인:

```
GET http://localhost:8003/channels
```

✅ 확인할 것:
- 방금 등록한 채널이 목록에 나오는지
- `active: true`, `keyword`, `url` 값이 올바른지

---

## Step 4. 채널 모니터 에이전트 테스트

**목적**: 스크래핑으로 새 영상을 감지하고 영상 큐 DB에 등록하는지 확인

```
POST http://localhost:8003/run/monitor
```

⏳ 백그라운드로 실행됩니다. 서버 콘솔 로그를 확인하세요.

### 로그에서 확인할 것:
```
═══ 채널 모니터 에이전트 시작 ═══
활성 채널 1개 조회
[1/1] 채널 스크래핑: 한국경제TV (키워드: 주식)
  ✓ 신규 등록: [영상 제목] (자막: Y)
═══ 채널 모니터 완료: {...} ═══
```

### Notion & API 확인:

```
GET http://localhost:8003/queue
```

✅ 확인할 것:
- 영상 큐 DB에 영상이 등록되었는지
- `subtitle_status`가 Y/N/미확인 중 하나인지
- `analysis_needed`가 "미정"인지
- `analysis_done`이 false인지

⚠️ 영상이 0개면:
- 키워드가 채널의 실제 영상 제목과 매칭되는지 확인
- 채널 URL이 올바른지 확인

---

## Step 5. 필터링 에이전트 테스트

**목적**: "분석필요=미정"인 영상에 대해 LLM이 분석 가치를 판단하는지 확인

> ⚠️ 이 에이전트는 07~20시에만 동작합니다. 시간 외에는 `skipped` 응답이 나옵니다.

```
POST http://localhost:8003/run/filter
```

### 로그에서 확인할 것:
```
═══ 필터링 에이전트 시작 ═══
필터링 대상: N건
  ✓ 분석 필요: [영상 제목]
  또는
  ✗ 분석 불필요: [영상 제목]
═══ 필터링 완료: {...} ═══
```

### Notion 확인:
- 영상 큐 DB에서 `분석필요` 컬럼이 "필요" 또는 "불필요"로 변경되었는지

✅ 확인할 것:
- LLM 호출이 정상 동작하는지 (API 키 오류 없는지)
- 10분 이하 영상이 자동으로 "불필요" 처리되는지

---

## Step 6. 보고서 생성 에이전트 테스트

**목적**: "분석필요=필요 & 자막=Y"인 영상의 자막을 LLM으로 분석하여 보고서 + 종목의견 생성

> 💡 Step 5에서 "필요"로 판정된 영상이 있어야 실행됩니다.
> 없으면 Notion에서 영상 큐의 `분석필요`를 직접 "필요"로, `자막상태`를 "Y"로 변경하세요.

```
POST http://localhost:8003/run/report
```

### 로그에서 확인할 것:
```
═══ 보고서 생성 에이전트 시작 ═══
보고서 생성 대상: N건
보고서 생성 중: [영상 제목]
  종목의견 N/N개 생성
  ✓ 보고서 생성 완료: [영상 제목] (종목 N개)
═══ 보고서 생성 완료: {...} ═══
```

### Notion & API 확인:

```
GET http://localhost:8003/reports
GET http://localhost:8003/opinions
```

✅ 확인할 것:
- **1차 보고서 DB**: 새 보고서 페이지가 생성되었는지, 본문에 분석 내용이 있는지
- **종목의견 DB**: 종목별 레코드가 생성되었는지
  - `원본_종목명`에 종목명이 있는지
  - `의견유형`이 추천/관심/주의 중 하나인지
  - `정규화_상태`가 "미처리"인지
- **영상 큐 DB**: 해당 영상의 `분석완료`가 ✅ (True)로 변경되었는지

---

## Step 7. 정규화 에이전트 테스트

**목적**: 종목의견 DB의 원본 종목명을 정규화 처리

> 💡 Step 6에서 종목의견이 생성된 후 실행합니다.

```
POST http://localhost:8003/run/normalize
```

### 로그에서 확인할 것:
```
═══ 정규화 에이전트 시작 (미처리 N건) ═══
  삼성전자 → 삼성전자 (완료)
  SK하이닉스 → SK하이닉스 (완료)
═══ 정규화 완료: {...} ═══
```

### Notion 확인:
✅ 확인할 것:
- `정규화_종목명`에 값이 채워졌는지
- `정규화_상태`가 "완료" 또는 "수동확인필요"로 변경되었는지

---

## Step 8. 전체 흐름 자동 실행 확인

**목적**: 스케줄러가 자동으로 에이전트들을 실행하는지 확인

1. 서버를 켜둔 채로 **10분** 대기
2. 서버 콘솔 로그에서 자동 실행 로그 확인:
   - 채널 모니터가 10분마다 실행되는지
   - 보고서 에이전트가 5분마다 실행되는지

✅ 별도의 수동 API 호출 없이 파이프라인이 돌아가면 성공!

---

## 트러블슈팅 체크리스트

| 증상 | 원인 | 해결 |
|------|------|------|
| `channels` API가 빈 배열 | Notion Integration 미연결 | Notion에서 DB → 연결(Connections) → Integration 추가 |
| 채널 모니터가 영상 0개 | 키워드 불일치 | 유튜브 채널 최근 영상 제목 확인 후 키워드 수정 |
| 필터링이 `skipped` | 07~20시 외 시간 | 해당 시간에 재실행 |
| LLM 호출 오류 | API 키 문제 | `.env`의 `LLM_API_KEY` 확인 |
| 보고서 생성 0건 | "필요" 영상 없음 | Notion에서 영상 큐 `분석필요`를 "필요"로 직접 변경 |
| Notion API 429 오류 | Rate limit | 자동 재시도 로직이 있으므로 대기 |
